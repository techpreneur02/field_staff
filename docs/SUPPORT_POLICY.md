# Support Policy

This policy defines what is included in buyer support for this module.

## 1. Included Support

- Installation guidance using provided documentation
- Assistance with configuration and initial setup
- Clarification of documented features and workflows
- Bug verification and confirmed bug fixes in supported versions

## 2. Not Included

- Custom feature development
- Third-party integration not documented in package scope
- Server administration unrelated to module behavior
- Data-entry services or payroll operations outsourcing

## 3. Support Prerequisites

When requesting support, provide:
- Module version
- Host application version
- PHP version
- Reproducible steps
- Screenshots and relevant error output
- Date/time and timezone context for payroll/attendance issues

## 4. Response Targets

- First response target: 1 to 2 business days
- Standard resolution target: depends on issue severity and reproducibility

## 5. Severity Guidance

- Critical: data loss, payroll blocking errors, access control bypass
- High: major workflow blocked without workaround
- Medium: partial workflow degradation with workaround
- Low: cosmetic defects and minor usability issues

## 6. Update Delivery

- Fixes and improvements are delivered in versioned releases
- All releases are documented in CHANGELOG.md

## 7. Best-Effort Limits

Support quality depends on environment parity and reproducible issue data.
